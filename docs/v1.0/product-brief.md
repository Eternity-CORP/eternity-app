# Product Brief: E-Y

## Executive Summary

**E-Y** is the first crypto wallet ecosystem designed to feel intuitive from day one. In a world where crypto remains "technology for techies," E-Y removes fear and complexity by bringing radical simplicity to blockchain — starting with bank-like UX and evolving into a comprehensive financial ecosystem.

**Core Innovations:**
- **BLIK-style codes** — world's first 6-digit code system for crypto transfers
- **Network Abstraction** — users see tokens, not chains
- **SHARD** — identity verification via NFC passport (World App Orb alternative)
- **AI Financial Agent** — proactive assistant with customizable personality

**Market:** $12-15B (2025), growing 25-30% CAGR. No competitor combines all four pillars.

**Vision:** *"E-Y — the crypto ecosystem that just works."*

---

## Core Vision

### Problem Statement

Crypto remains "technology for techies" — regular people are **afraid** to use it.

This isn't just inconvenience, it's **fear**:
- Fear of sending to the wrong network and losing everything forever
- Fear of not understanding interfaces filled with unfamiliar terms
- Fear that "crypto isn't for me"

**Root pain:** *"Crypto scares regular people away."*

### Problem Impact

| Who suffers | How it manifests |
|-------------|------------------|
| **Newcomers (Oksana)** | Want to try crypto but fear losing money |
| **Recipients (Andrey)** | Client wants to pay in crypto — forced to figure out addresses |
| **Even power users (Max)** | Waste time choosing networks, verifying addresses |

**Consequence:** Mass crypto adoption is blocked not by technology, but by UX. People stay with traditional banks even though they want control over their money.

### Why Existing Solutions Fall Short

| Competitor | Problem |
|------------|---------|
| **MetaMask** | Built for developers, intimidates regular users |
| **Trust Wallet** | Many features, no UX innovation |
| **World App** | Requires finding an Orb for verification (~2000 worldwide) |
| **Phantom** | Great UX, but locked into Solana ecosystem |
| **TON Wallet** | TON chain only, no EVM support |

**Key gap:** No one has brought intuitive UX to crypto. Everyone adds features, no one rethinks the experience.

### Proposed Solution

**E-Y** solves the problem through 4 pillars:

1. **BLIK-style codes for crypto** (world's first)
   - 6-digit code instead of address
   - Valid for 2 minutes
   - "Give code — receive money" — like Polish BLIK

2. **Network Abstraction**
   - User sees "USDC", not "USDC (Polygon)"
   - System auto-selects optimal route
   - Pro mode for those who need control

3. **SHARD Identity** (post-MVP)
   - NFC passport → unique crypto identity
   - Scan at home, don't hunt for Orbs
   - 1.4B+ NFC passports vs ~2000 Orbs globally

4. **AI Financial Agent**
   - Proactive notifications ("Gas is cheap — time for that swap!")
   - Customizable personality (professional, friendly, etc.)
   - Full financial management via chat

**The "aha!" moment:** *"Wait... I just gave 6 digits and the money arrived? No addresses? This just works!"*

### Key Differentiators

| Differentiator | Why it matters | Why it's hard to copy |
|----------------|----------------|----------------------|
| **BLIK-first** | No one does this — first mover | Requires complete UX rethinking |
| **Ecosystem thinking** | Not just wallet — evolving platform | Requires long-term vision & discipline |
| **SHARD vs Orb** | Accessibility: passport vs physical device | Regulatory knowledge + NFC expertise |
| **Proactive AI** | Not a chatbot, a financial agent | Requires deep integration |

**Why now (2026):**
- Post-FTX: people want self-custody without complexity
- Chain abstraction tech is mature (Particle, Abstract)
- AI became affordable (GPT-4o-mini, Whisper)
- EU eIDAS 2.0 legalized NFC passport verification for private entities

**Unique superpower:** Understanding that crypto doesn't need another wallet with features — it needs the first wallet ecosystem that just works. And the discipline to ship it.

### Evolution Path

```
Phase 1: Bank-like UX          →  Phase 2: Intelligence    →  Phase 3: Identity & Scale
(Simple transfers, BLIK)          (AI Agent, Proactive)       (SHARD, Multi-chain, Cards)
         ↓                                 ↓                            ↓
    "It just works"              "It thinks for me"           "It's my crypto life"
```

E-Y is not a destination — it's an evolving ecosystem that grows with its users.

## MVP Scope

### Core Features (Must Have)

#### Wallet Core
- Create wallet (seed phrase with clear explanation)
- Import wallet (seed phrase)
- Multi-account support (1 seed → multiple accounts)
- View balances (tokens + USD equivalent)

#### Transfers
- Send to address (standard EVM transfer)
- Send to @username (centralized mapping)
- Receive (address, QR code, @username)
- **BLIK Code Payment** — 6-digit codes for instant P2P transfers (killer feature)

#### UX Foundation
- Network abstraction (users see tokens, not chains)
- Token-first UI ("USDC" not "USDC Polygon")
- Prepared design (polished UI, not placeholder)

#### Feature Overlays (Built on Send)
- Contact Book (save frequent recipients)
- Scheduled Payments (recurring transfers)
- Split Bill (request payments from multiple people)

### Out of Scope (Post-MVP)

| Feature | Rationale |
|---------|-----------|
| **SHARD Identity** | Requires NFC SDK integration, regulatory research |
| **AI Financial Agent** | Add after core wallet is stable |
| **Cross-chain bridges** | Same-chain first, cross-chain later |
| **Swap integration** | After basic transfers work |
| **Pro mode** | Default simplicity first |
| **Virtual cards** | Future ecosystem expansion |
| **Decentralized username** | Centralized MVP → decentralized migration later |

### MVP Success Criteria

| Criterion | Definition of Success |
|-----------|----------------------|
| **Works on testnet** | All features functional on Ethereum Sepolia |
| **BLIK demo ready** | Can demonstrate at EthCC, get "wow" reactions |
| **10 beta testers** | Real people using without assistance |
| **No critical bugs** | Funds safe, UX doesn't break |
