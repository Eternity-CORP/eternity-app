## Part 2: Unique Features Validation

### 2.1 BLIK-Style Crypto Payments

#### What is BLIK?
BLIK is a Polish mobile payment system where users generate 6-digit codes valid for 2 minutes to authorize payments. 80%+ of Polish smartphone users use BLIK.

#### Crypto Market Analysis

**CRITICAL FINDING: NO CRYPTO-NATIVE BLIK EQUIVALENT EXISTS**

| Searched For | Result |
|--------------|--------|
| "BLIK crypto wallet" | No results |
| "6-digit code crypto payment" | No relevant products |
| "temporary code P2P crypto" | No equivalent systems |
| "ephemeral payment code blockchain" | Only traditional 2FA |

**Competitors' Approach to P2P:**
- MetaMask: Address or ENS
- Trust Wallet: Address or scan QR
- Phantom: Address or Solana Name Service
- All require recipient to share address or QR code

**E-Y Opportunity**:
- First crypto wallet with BLIK-style codes
- Solves "I don't have my address handy" problem
- Perfect for in-person payments without showing QR
- Could become industry standard

### 2.2 Network/Chain Abstraction

#### Market Landscape

| Solution | Approach | Status |
|----------|----------|--------|
| **Particle Network** | Universal Accounts - one address across chains | Production (2025) |
| **Abstract (zkSync)** | Global Wallet with social recovery | Growing ecosystem |
| **NEAR Chain Signatures** | MPC-based cross-chain signing | Live on mainnet |
| **Xion** | Native chain abstraction L1 | Launched late 2024 |
| **Coinbase Smart Wallet** | Simplified UX for Base chain | Limited scope |

**Technical Approaches:**
1. **Intent-based**: User expresses intent, solver finds best route
2. **MPC-based**: Multi-party computation for cross-chain signatures
3. **Account Abstraction (ERC-4337)**: Smart contract wallets with flexible auth
4. **Unified liquidity layers**: Abstract away bridging complexity

**E-Y Strategy Validation**:
- Focus on UX abstraction (hide networks from users)
- Use proven bridges (LI.FI, Rango) for execution
- Start with EVM chains, expand later
- This is the right approach for "bank-like" experience

### 2.3 SHARD Identity System (NFC Passport)

#### Technology Maturity

| Aspect | Status |
|--------|--------|
| NFC Chip in Passports | 150+ countries support |
| EU eIDAS 2.0 | Mandates digital identity by 2026 |
| EU Regulation 2025/1208 | Private entities can now verify ePassports |
| SDK Availability | Multiple production-ready options |

#### Available SDKs

| Provider | Features | Pricing |
|----------|----------|---------|
| **ReadID (Inverid)** | Full NFC + face match | Enterprise |
| **Persona** | Identity verification suite | Usage-based |
| **Didit** | Web3-focused, privacy-preserving | Free tier available |
| **Regula** | Document + NFC verification | Enterprise |

#### Comparison with World App Orb

| Aspect | World App Orb | E-Y SHARD (NFC) |
|--------|---------------|-----------------|
| Accessibility | ~2000 Orbs globally | 1.4B+ NFC passports |
| Privacy | Iris scan stored (concerns) | Cryptographic proof only |
| User Action | Visit Orb location | Scan at home |
| Cost | High (Orb hardware) | Low (phone NFC) |
| Uniqueness | Very strong | Strong (passport = 1 person) |

**E-Y Advantage**: SHARD offers similar uniqueness guarantee with dramatically better accessibility. Users can verify at home instantly.
