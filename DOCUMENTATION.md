## 💡 Documentation Guidelines

### Where to Add New Docs

**Mobile Features:**
- Add to `mobile/docs/features/`
- Update `mobile/docs/README.md`

**Guides & Tutorials:**
- Add to `mobile/docs/guides/` or `docs/`
- Update relevant README

**API Changes:**
- Update `docs/api/API_DOCUMENTATION.md`
- Update backend docs if needed

**Architecture Changes:**
- Update `docs/architecture/ARCHITECTURE.md`
- Update SRS if requirements changed

### Documentation Standards
- Use Markdown format
- Include code examples
- Add table of contents for long docs
- Keep README files updated
- Use emojis for better navigation 📚
